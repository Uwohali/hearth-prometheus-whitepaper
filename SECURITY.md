# Security Policy

## Reporting

Report suspected security, provenance, release-integrity or credential issues privately to the repository owner.

Do not include credentials, private keys, tokens, personal data or unreleased evidence in public issues.

## Scope

This repository is a publication mirror. Runtime vulnerabilities belong in the relevant implementation repository, including:

- `Permaculture-DAO/prometheus-happ`
- `Permaculture-DAO/prometheus-runtime`

## Release-integrity incidents

A suspected manifest, checksum, signature or provenance failure must trigger:

1. publication freeze;
2. status downgrade;
3. affected-claim review;
4. correction or withdrawal;
5. documented re-verification.
